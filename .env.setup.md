# Guía Configuración Variables .env

## 1. Evolution API

Obtener datos de tu instancia Evolution:

```env
EVOLUTION_API_URL=https://evo.lab7.com.ar
EVOLUTION_API_TOKEN=1FAECBD2909F-48D4-AA03-B5287BA7CF68
EVOLUTION_INSTANCE=Facturas-WhatsApp
```

**Dónde obtenerlo:**
1. Abrir Evolution dashboard
2. Seleccionar instancia `Facturas-WhatsApp`
3. Copiar:
   - API URL base
   - API Token (mostrado en configuración)
   - Nombre exacto instancia

## 2. Chatwoot

CHATWOOT_URL=https://chat.lab7.com.ar
CHATWOOT_WEBHOOK_SECRET=9Hnte4Jkk4M1c5ozoMCUfHtL

2. Copiar:

## 3. Mercado Pago

```env
MP_ACCESS_TOKEN=APP_USR-xxxxxxxxxxxxxxxxxxxx-xxxxxx
MP_PLAN_ID=plan_123456
MP_WEBHOOK_SECRET=tu_webhook_secret
```

**Pasos:**
1. Crear cuenta en https://www.mercadopago.com.ar
2. Ir a Settings → Credentials
3. Copiar Access Token
4. Crear plan de suscripción:
   - Nombre: "Facturas7"
   - Monto: $500
   - Frecuencia: Mensual
   - Copiar Plan ID

## 4. Afip (configuración futura)

```env

AFIPSDK_ENTORNO=homologacion

AFIP_CERTIFICADO_PATH=./src/afip/certificado.pem
```

**Pasos:**
1. Obtener certificado digital de CUIT en Afip
2. Guardar archivo `.pem` en `/src/afip/`
3. Configurar en .env

## 7. Verificación

Testear que todo esté conectado:

```bash
# 1. Backend arranca sin errores
npm start

# 2. Verificar logs
# Debe mostrar:
# - BD inicializada
# - Servidor en puerto 3000
# - Evolution/Chatwoot credenciales OK (si están configuradas)
```

## 8. Producción (Railway/Vercel)

1. Crear proyecto Railway
2. Agregar variables:
   ```
   railway variables add EVOLUTION_API_URL=...
   railway variables add EVOLUTION_API_TOKEN=...
   ... (todas las variables)
   ```
3. Deploy:
   ```
   railway up
   ```
